## [0.4.0](https://github.com/bancsdan/go-stripenav/compare/v0.3.0...v0.4.0) (2026-06-11)

Highlights beyond the commit list below:

- **Submissions are never abandoned.** Crossing NAV's 24-hour reporting
  window no longer aborts a row — late reporting is still legally
  required and beats never reporting. The worker keeps retrying and
  emits a Warn log plus a `deadline_exceeded` metric event;
  `StatusAborted` now only arises when a storno's parent CREATE
  terminally failed.
- **Webhook ACK semantics.** Transient failures (store unavailable,
  exchange-rate lookup failed) return 503 so Stripe redelivers;
  permanent ones (undecodable / unmappable events) are ACKed with 200
  and logged. Previously every failure was ACKed, silently losing
  events that were never persisted.
- **No duplicate submissions on flaky polls.** Retriable
  `queryTransactionStatus` failures back off in place, keeping the NAV
  `transactionId`, instead of requeueing the row for a second submit.
- **Store contract additions.** `SubmissionStore.Put` should wrap the
  new `stripenav.ErrAlreadyExists` for duplicate event ids;
  `ClaimBatch` must not hand back rows whose lease is still valid, even
  to the holder. The bundled in-memory store implements both.
- **Breaking:** `WorkerConfig.Supplier` was removed (it was never
  read), and the NAV client, mapper internals, and in-memory store
  moved behind `internal/` — the public surface is the root package
  plus `nav`, `mapping`, and `nav/schemas`.

## 0.3.0 and earlier (2026-05 – 2026-06)

Initial development, released as v0.1.0–v0.3.0 before changelog
automation:

- `stripenav.Handler` — the embeddable `http.Handler` Hungarian Stripe
  merchants mount on their backend — and `stripenav.Config`, the single
  struct for Stripe-side, NAV-side, supplier, store, exchange-rate,
  logging, metrics, and clock configuration.
- Stripe `Invoice` → NAV `InvoiceData` translation: Hungarian
  VAT-number splitting, customer-category classification
  (DOMESTIC / OTHER / PRIVATE_PERSON), `big.Rat`-based monetary
  aggregation with HUF summary rendering, and typed `*MappingError`
  results.
- NAV Online Számla v3.0 client covering `tokenExchange` (AES-128-ECB
  token decryption; tokens are single-use, deliberately uncached),
  `manageInvoice`, `manageAnnulment`, `queryTransactionStatus` — with
  SHA-512 password hash, SHA3-512 request signing, typed `*nav.NAVError`
  responses with retriability classification, and a `MaxBatchSize`
  guard.
- `Submission` state machine (pending → submitted → processing →
  accepted / rejected / aborted), the `SubmissionStore` interface with
  claim/lease semantics, and an in-memory reference store.
- `Worker` driving retries with exponential backoff (±20% jitter),
  claim/lease-based multi-replica safety, and parent-dependency
  tracking for stornos; pluggable `MetricsRecorder` and `Clock` hooks.
- `docs/EMBED.md` integration guide; e2e suite against the real NAV
  test environment (`-tags=navtest`); race-clean under
  `go test ./... -race`.
